AWS ECR Dynamo Container Registry Access

Registry210086341041.dkr.ecr.us-west-2.amazonaws.com/ai-dynamo/dynamo
Architecturesamd64, arm64
Auth methodnvsec (nvsec authentication guide)
Also available viaAzure ACR (amd64 only, simpler auth)

Prerequisites (one-time)

Install AWS CLI and nvsec. See nvsec authentication guide.

pip install nvsec
aws configure  # or use nvsec for SSO

Step 1: Login

# Authenticate to AWS via nvsec (opens browser if needed)
nvsec login

# Docker login to ECR (12-hour token)
aws ecr get-login-password --region us-west-2 \
  | docker login --username AWS --password-stdin \
    210086341041.dkr.ecr.us-west-2.amazonaws.com

Step 2: List available images

# List tagged images
aws ecr list-images \
  --registry-id 210086341041 \
  --repository-name ai-dynamo/dynamo \
  --region us-west-2 \
  --filter tagStatus=TAGGED \
  --query 'imageIds[*].imageTag' \
  --output table

Step 3: Pull an image

Image tags follow the pattern: <commit_sha>-<framework>-<type>-cuda<ver>-<arch>

# Pull a vLLM dev image (amd64, CUDA 12)
ECR=210086341041.dkr.ecr.us-west-2.amazonaws.com/ai-dynamo/dynamo
docker pull $ECR:<commit_sha>-vllm-dev-cuda12-amd64

# Pull an SGLang runtime image (arm64, CUDA 13)
docker pull $ECR:<commit_sha>-sglang-runtime-cuda13-arm64

Step 4: Pull + build local-dev

(ECR=210086341041.dkr.ecr.us-west-2.amazonaws.com/ai-dynamo/dynamo:<tag> \
 && docker pull $ECR \
 && curl -sL http://speedoflight.nvidia.com/dynamo/dynamo-utils/container/build_localdev_from_dev.py \
    | python3 - --skip-pull $ECR)

Tag format reference

ComponentValues
commit_sha40-character git commit SHA
frameworkvllm, sglang, trtllm, dynamo, none
typedev, runtime, test, efa
cudacuda12, cuda13 (maps to 12.9, 13.x)
archamd64, arm64
